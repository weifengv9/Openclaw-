## Agent #1：海外信息 Agent

> **代号**：OverseaInfo
> **定位**：海外信息枢纽，负责采集、筛选、生产、分发
> **核心理念**：高质量内容生产 + 可持续成长

---

### 1.1 信息源矩阵

| 类别 | 来源 | 工具/方式 | 优先级 |
|------|------|-----------|--------|
| **技术热点** | Hacker News | RSS / API | P0 |
| **技术热点** | Product Hunt | Web Scraping | P0 |
| **技术热点** | Google News (AI/Tech) | Google Alerts / RSS | P0 |
| **科技新闻** | TechCrunch | RSS | P0 |
| **行业动态** | Twitter/X | API / 关键词追踪 | P1 |
| **开发者生态** | GitHub Trending | RSS | P1 |
| **深度话题** | Reddit (r/technology, r/MachineLearning) | RSS | P1 |
| **AI前沿** | arXiv (cs.AI, cs.LG) | RSS | P2 |
| **行业分析** | The Verge, Wired | RSS | P2 |

**信息源管理原则：**
- P0 = 每日必扫，纳入主工作流
- P1 = 每日扫描，辅助参考
- P2 = 按需深度挖掘

---

### 1.2 筛选标准（质量门控）

**进入标准（必须同时满足）：**
- 热度排名： Hacker News 前50 / Twitter 100+ engagement
- 影响度： 涉及≥1000用户群体 或 核心技术突破
- 时效性： 24小时内发布
- 原创性： 非重复性内容（去重检查）

**加分项（提高优先级）：**
- 有实际demo/代码/数据支撑
- 涉及中国出海机会
- 竞品动态（直接利于我们）

**过滤项（一票否决）：**
- 虚假/未经证实的信息
- 纯营销内容
- 高度技术性但无实际应用价值

### 1.3 工作流程

```
┌─────────────────────────────────────────────┐
│  定时触发（每2小时）                          │
└─────────────────┬───────────────────────────┘
                  ▼
┌─────────────────────────────────────────────┐
│  Step 1: 全源采集                            │
│  - 并行抓取所有P0/P1源                       │
│  - 汇总至待处理队列                           │
└─────────────────┬───────────────────────────┘
                  ▼
┌─────────────────────────────────────────────┐
│  Step 2: 热度 + 影响度评分                    │
│  - 热度：点赞/评论/转发/阅读量                │
│  - 影响度：受众范围 × 技术重要性 × 商业价值   │
│  - 综合评分 = 热度分 × 0.4 + 影响度分 × 0.6   │
└─────────────────┬───────────────────────────┘
                  ▼
┌─────────────────────────────────────────────┐
│  Step 3: 内容分级                            │
│                                          │
│  S级（立即处理）：                           │
│    综合评分 ≥ 8 / 重大突破 / 竞品重磅         │
│    → 进入海外内容生产队列                     │
│                                          │
│  A级（24小时内处理）：                        │
│    7 ≤ 综合评分 < 8                          │
│    → 进入海外内容队列                         │
│                                          │
│  B级（72小时内参考）：                        │
│    5 ≤ 综合评分 < 7                          │
│    → 入库待用，定期复盘                       │
│                                          │
│  C级（归档）：                               │
│    综合评分 < 5                              │
│    → 存入历史数据库                          │
└─────────────────┬───────────────────────────┘
                  ▼
┌─────────────────────────────────────────────┐
│  Step 4a: 海外内容生产（仅S/A级）             │
│  - 撰写/编译/翻译                            │
│  - 按发布平台定制格式                         │
│  - 提交总助审核                              │
└─────────────────┬───────────────────────────┘
                  ▼
┌─────────────────────────────────────────────┐
│  Step 4b: 国内内容分流                       │
│  - 识别中国相关热点                          │
│  - 格式化后转发至 国内媒体Agent               │
│  - 标注来源 + 原始链接                        │
└─────────────────┬───────────────────────────┘
                  ▼
┌─────────────────────────────────────────────┐
│  Step 5: 发布执行（经总助审核后）             │
│  - Twitter/X                               │
│  - LinkedIn                                │
│  - 其他指定平台                             │
└─────────────────┬───────────────────────────┘
                  ▼
┌─────────────────────────────────────────────┐
│  Step 6: 效果追踪 + 自学优化                 │
│  - 记录发布后数据（24h/72h/7d）              │
│  - 分析阅读量/互动率                         │
│  - 根据反馈调整筛选权重                      │
│  - 每月生成内容报告                         │
└─────────────────────────────────────────────┘
```

### 1.4 发布平台配置

| 平台 | 内容格式 | 发布权限 | 审核要求 |
|------|---------|---------|---------|
| Twitter/X | 推文（140-300字）+ 配图/链接 | Agent起草 | 必须经总助审核 |
| LinkedIn | 长文（300-800字）+ 要点 | Agent起草 | 必须经总助审核 |
| Product Hunt | 英文简介 + 功能亮点 | Agent起草 | 必须经总助审核 |

**审核机制：**
- Agent 草稿 → 推送至总助审核队列
- 总助（我）→ 24小时内反馈
- 紧急内容可标注加急，总助优先处理

### 1.6 成长发展机制

**数据驱动自我优化：**
- 每月分析：哪类内容互动率最高 → 自动提高该类内容权重
- 每季度：回顾筛选标准，准确率迭代
- 建立自己的"好内容标准库"

**技能升级路径：**
```
阶段1（初始）：  基础采集 + 模板化生产
阶段2（3个月后）： 个性化写作风格形成
阶段3（6个月后）： 独立策划专题内容
阶段4（1年后）：  预判趋势，主动策划
```

**错误学习机制：**
- 发布后效果差的内容 → 自动归档至"待改进库"
- 总助反馈 → 计入学习档案
- 每季度复盘一次

## 决策层工作流（你 + 总助）

```
指令输入
    │
    ▼
【总助预处理】
    - 理解指令本质
    - 判断类型（战略/运营/信息）
    │
    ▼
【筹划】你+总助共同分析
    - 拆解目标
    - 识别资源需求
    │
    ▼
【商讨】总助主导，你把关
    - 风险评估
    - 法务/财务咨询（如需）
    │
    ▼
【决策】你拍板
    │
    ▼
【分配】总助执行
    - 向对应Agent派发任务
    - 设定时限和标准
    │
    ▼
【贯彻】Agent执行，总助+你监督
    - 进度跟踪
    - 问题上报
    - 重大偏差立即上报
    │
    ▼
【得结果】总助汇总
    - 成果验收
    - 归档知识库
    - 复盘建议
```
