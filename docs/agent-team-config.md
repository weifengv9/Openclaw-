# Agent 团队配置

> 7步决策流 Agent 团队 + 法务/财务支撑
> 基于"总办 → 筹划 → 商讨 → 决策 → 分配 → 贯彻 → 得结果"架构

---

## 团队成员

| # | Agent | 职责 | 工具配置 |
|---|-------|------|----------|
| 1 | **总办** (Executive) | 接收指令、统筹协调、派发任务、最终验收 | 全部工具 |
| 2 | **筹划** (Planner) | 制定方案、拆解路径、资源规划 | exec, read, write |
| 3 | **商讨** (Deliberator) | 评估风险、论证可行性、征求意见 | exec, read, tavily_search |
| 4 | **决策** (Decider) | 分析利弊、拍板定论、明确结论 | exec, read |
| 5 | **分配** (Allocator) | 拆解任务、明确责任人、时间节点 | exec, read, write |
| 6 | **贯彻** (Executor) | 执行跟踪、进度同步、问题上报 | exec, read, sessions_send |
| 7 | **DocGardener** (知识库) | 文档沉淀、知识整理、归档维护 | read, write, memory_search |
| 8 | **法务** (Legal) | 法律风险审查、合规审核 | Skill: `legal-review` |
| 9 | **财务** (Finance) | 财务可行性审查、预算审核 | Skill: `finance-review` |

---

## 决策流工作流程

```
用户指令
    │
    ▼
┌──────────────────────────────────────┐
│ #1 总办 — 接收 + 分析 + 分派           │
└──────┬───────────────────────────────┘
       │ 派发任务
       ▼
┌──────────────────────────────────────┐
│ #2 筹划 — 制定初步方案                 │
└──────┬───────────────────────────────┘
       │ 提交方案
       ▼
┌──────────────────────────────────────┐
│ #3 商讨 — 评估风险 + 征求意见           │
│   （可咨询 #8 法务 / #9 财务）         │
└──────┬───────────────────────────────┘
       │ 审议通过
       ▼
┌──────────────────────────────────────┐
│ #4 决策 — 拍板定论                     │
└──────┬───────────────────────────────┘
       │ 决策通过
       ▼
┌──────────────────────────────────────┐
│ #5 分配 — 拆解任务 + 明确责任人         │
└──────┬───────────────────────────────┘
       │ 分配就绪
       ▼
┌──────────────────────────────────────┐
│ #6 贯彻 — 执行 + 跟踪 + 上报问题        │
└──────┬───────────────────────────────┘
       │ 完成
       ▼
┌──────────────────────────────────────┐
│ #7 得结果 — 验收 + 归档知识库           │
│        + DocGardener 沉淀文档          │
└──────────────────────────────────────┘
```

## 四个验收维度

| 维度 | 含义 | 最低达标线 |
|------|------|-----------|
| **质量** | 输出准确、无错误、完整 | 90% 正确率 |
| **原创性** | 有独特价值、不重复 | 与库内内容重复率 < 20% |
| **工艺** | 专业程度、结构清晰 | 符合文档规范 |
| **功能** | 满足需求、可落地 | 核心功能 100% 实现 |

## Agent 间通信

通过 `sessions_send` 工具进行 Agent 间消息传递。

消息格式：
```json
{
  "sessionKey": "agent:planner:default",
  "message": "任务描述 + 验收标准 + 上下文"
}
```

## DocGardener 知识库管理规范

### 扫描周期
- **高频**：每周一次（周三早上 9:00）
- **低频**：每月一次全面检查

### 扫描范围
| 目录 | 优先级 | 说明 |
|------|--------|------|
| `docs/knowledge/decisions/` | 高 | 决策记录，需及时归档 |
| `docs/knowledge/plans/` | 高 | 筹划方案，需跟踪更新 |
| `docs/knowledge/lessons/` | 高 | 经验教训，需持续沉淀 |
| `docs/planning/` | 高 | 计划文档需及时更新 |
| `memory/` | 中 | 日常记录，定期整理 |

### 更新标准
| 问题类型 | 处理方式 |
|----------|----------|
| 过时信息 | 更新为最新内容 |
| 缺失内容 | 补充完善 |
| 结构混乱 | 重新整理结构 |
| 重复内容 | 合并或删除 |
| 错误内容 | 修正或删除 |
