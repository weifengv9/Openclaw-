# Generator Agent 验收标准

> Generator = 生成器，负责内容创建、代码编写、页面生成

---

## 输出标准

### 1. 质量标准
- [ ] 内容准确，无事实性错误
- [ ] 代码可运行，无语法错误
- [ ] 符合既定的格式规范
- [ ] 无错别字或表达不清的地方

### 2. 原创性标准
- [ ] 避免照抄模板，有自己的理解和表达
- [ ] 引用资料时标注来源
- [ ] 给出独特的见解或解决方案

### 3. 工艺标准
- [ ] 结构清晰，分层合理
- [ ] 命名规范，代码风格统一
- [ ] 有适当的注释和说明
- [ ] 文件存放到正确路径（见路径规范）

### 4. 功能标准
- [ ] 满足用户需求的核心功能
- [ ] 边界情况有处理
- [ ] 有基本的可扩展性

---

## 生成器自查清单

```
□ 任务理解正确吗？
□ 我知道用户在哪干活、用什么干活、怎么知道对不对吗？
□ 我需要什么信息？去哪个文件夹找？
□ 生成的内容有检查过质量吗？
□ 符合四个维度的标准吗？
□ 出错了，知道哪里不对、怎么改吗？
```

## 输出位置

| 类型 | 路径 |
|------|------|
| 设计文档 | `docs/design/` |
| 架构文档 | `docs/architecture/` |
| 计划文档 | `docs/planning/` |
| 研究资料 | `docs/research/` |
| 模板文件 | `docs/templates/` |

---

## 错误修正流程

当 Evaluator 返回修正意见时：
1. 仔细阅读修正意见
2. 定位问题位置
3. 修改对应内容
4. 重新提交审查
