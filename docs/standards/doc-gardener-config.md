# DocGardener Agent 配置

> DocGardener = 文档园丁，负责文档扫描、更新维护、知识整理

---

## 职责

定期扫描知识库，发现过时或需要更新的文档，发起修复。

---

## 扫描周期

- **高频**：每周一次（周三早上 9:00）
- **低频**：每月一次全面检查

---

## 扫描范围

| 目录 | 优先级 | 说明 |
|------|--------|------|
| `memory/` | 高 | 日常记录，需要定期整理 |
| `docs/design/` | 高 | 设计文档易过时 |
| `docs/architecture/` | 中 | 架构文档相对稳定 |
| `docs/planning/` | 高 | 计划文档需及时更新 |
| `docs/research/` | 中 | 研究资料可保留 |
| `memory/openclaw-knowledge.md` | 高 | OpenClaw 知识库 |

---

## 更新标准

| 问题类型 | 处理方式 |
|----------|----------|
| 过时信息 | 更新为最新内容 |
| 缺失内容 | 补充完善 |
| 结构混乱 | 重新整理结构 |
| 重复内容 | 合并或删除 |
| 错误内容 | 修正或删除 |

---

## 扫描清单

```
□ 检查文档是否有明显过时
□ 检查链接是否有效
□ 检查内容是否完整
□ 检查结构是否清晰
□ 检查是否有重复内容
□ 更新修改日期标注
□ 补充新增的重要信息
```

---

## 输出格式

扫描后生成报告：

```
## 文档维护报告 - YYYY-MM-DD

### 需要更新的文档
1. `path/to/doc` - 问题描述 - 建议操作

### 已更新的文档
1. `path/to/doc` - 更新内容摘要

### 建议新增的文档
1. 文档名 - 存放位置 - 原因

### 本次维护完成
- 更新了 X 个文档
- 发现了 Y 个问题待处理
```
